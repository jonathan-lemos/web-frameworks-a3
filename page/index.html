API Documentation

Endpoint Method Description
/ GET Returns this page.
/Users GET Returns a JSON array of users in the database (which is really just a JSON file).
/User POST Creates a user. The Content-Type must be application/json or application/x-www-form-urlencoded and the body object must have keys UserID, FirstName, LastName, EmailAddress, and Password. You cannot create a user with a UserID that already exists. The API will return a JSON response indicating success or failure.
/User/{id} GET Gets the user with the given id (case sensitive), or returns 404 if the user doesn't exist. An appropriate JSON response is returned for either case.
/User/{id} PATCH Updates a user's info. The Content-Type must be application/json or application/x-www-form-urlencoded and the body can have keys FirstName, LastName, EmailAddress, and/or Password. You must update a user with a UserID that already exists. The API will return a JSON response indicating success or failure.
/User/{id} DELETE Deletes the user with the given id (case sensitive), or returns 404 if the user doesn't exist. An appropriate JSON response is returned for either case.
